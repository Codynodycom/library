# Пошаговая инструкция по настройке VSCode для работы с GitHub и Git

## Установка расширений для работы с Git и GitHub в VSCode

1. Откройте VSCode.
2. Нажмите на иконку Extensions (Расширения) в левой боковой панели или используйте комбинацию клавиш Ctrl+Shift+X.
3. В строке поиска введите Git Graph.
4. Найдите расширение Git Graph и нажмите Install, чтобы установить его.

## Авторизация в GitHub через VSCode

1. Откройте **VSCode** и нажмите на иконку **Account** в левом нижнем углу окна.
2. Выберите **Sign in to GitHub...**.
3. В браузере откроется страница для авторизации GitHub. Введите свои учетные данные GitHub и предоставьте доступ Visual Studio Code.
4. После успешной авторизации вернитесь в **VSCode**.

## Клонирование репозитория через Source Control в VSCode

1. В **VSCode** нажмите на иконку **Source Control** (управление версиями) в левой боковой панели или используйте комбинацию клавиш `Ctrl+Shift+G`.
2. Нажмите на кнопку **Clone Repository** (Клонировать репозиторий).
3. Вставьте URL репозитория GitHub через HTTPS:
   ```plaintext
   https://github.com/username/repository-name.git
    ```
4. Выберите папку, куда вы хотите сохранить репозиторий, и дождитесь завершения клонирования.
5. Когда клонирование завершится, VSCode предложит открыть репозиторий. Нажмите Open.

## Использование Source Control для работы с Git

### Просмотр изменений:
- В разделе **Source Control** вы можете отслеживать изменения в файлах. Все измененные файлы будут отображаться в списке.

### Добавление файлов в индекс:
- Чтобы добавить файлы в индекс для коммита, нажмите на значок `+` рядом с каждым файлом.

### Создание коммита:
- После добавления файлов в индекс, введите сообщение коммита в поле в верхней части панели **Source Control** и нажмите на иконку галочки `✔` для создания коммита.

### Отправка изменений на GitHub:
- Для отправки изменений на удаленный репозиторий нажмите на иконку с тремя точками (в правом верхнем углу панели **Source Control**) и выберите **Push**.

### Получение обновлений с удаленного репозитория:
- Чтобы обновить локальный репозиторий, выберите **Pull** из того же меню с тремя точками.

## Использование Git Graph для визуализации работы с Git

1. Откройте **Command Palette** (Командная палитра) с помощью комбинации клавиш `Ctrl+Shift+P`.

2. Введите **Git Graph: View Git Graph** и выберите эту команду.

3. В новой вкладке откроется визуализация вашего репозитория:
   - Визуальное представление всех коммитов и веток.
   - Возможность просматривать подробности коммитов и переключаться между ветками.
   - Возможность делать слияние веток (merge) и откаты изменений (reset).

4. В интерфейсе **Git Graph** вы можете:
   - Просматривать и анализировать историю коммитов.
   - Выполнять операции над ветками, такие как создание новых веток или слияние.

## Открытие терминала и выполнение команд Git

1. **Открытие встроенного терминала в VSCode:**
   - Нажмите `Ctrl+Shift+`` (обратная кавычка) или перейдите в меню **Terminal** → **New Terminal**.

2. **Выполнение команд Git:**
   - В открывшемся терминале вы можете выполнять команды Git напрямую. Например:
     ```bash
     git status
     git add .
     git commit -m "Ваше сообщение коммита"
     git push
     ```
   - Каждая команда выполняет определенную функцию:
     - `git status` — отображает состояние вашего репозитория, включая измененные и неотслеживаемые файлы.
     - `git add .` — добавляет все измененные и новые файлы в индекс для коммита.
     - `git commit -m "Сообщение"` — фиксирует изменения с указанным сообщением.
     - `git push` — отправляет ваши коммиты на удаленный репозиторий.

Теперь вы готовы использовать **VSCode** для работы с **Git** и **GitHub**, используя как графический интерфейс **Source Control**, так и встроенный терминал для выполнения команд.
